Thread note bodies use the same markdown parser and DOM renderer as card bodies so agent replies can contain paragraphs, lists, tables, code blocks, inline code, horizontal rules, markdown headings, and bold spans. Default note text is regular weight Ubuntu text, agent replies stay left as unframed conversation text, operator notes stay right in a neutral grey card with only a left target-accent border, metadata is reduced to status only, and the thread input has no CSS border, only a lighter neutral inset surface.
