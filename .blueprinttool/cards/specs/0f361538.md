Card description content is a first-class file editing surface. Migrated ledgers store card bodies in `.blueprinttool/cards/<ledger-name>/<card-id>.md` and keep only `comment.contentFile` in the ledger JSON.

Runtime ledger reads hydrate `comment.what` from the sidecar before returning data to the browser, while edits from the browser and `ledger-cli` write back to the sidecar instead of embedding the full body in JSON.
