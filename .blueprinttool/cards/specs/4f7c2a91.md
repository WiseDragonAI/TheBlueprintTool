`generator-cli` is reserved for scaffold generation from the MasterLedger and related non-mutating ledger checks. Committed ledger JSON inspection, mutation, and markdown export must be owned by the separate `ledger-cli` package and executable, so editing specs/data ledgers cannot be coupled to scaffold generation control flow. `ledger-cli export --ledger <file> --output <file.md>` must write a complete markdown export grouped by canvas zones sorted by x then y, with zone titles as sections and card titles plus card content as subsections.

For migrated ledgers, `ledger-cli` must treat `comment.contentFile` as the authoritative card body storage. `ledger-cli export` hydrates sidecar Markdown into the exported document, and targeted card comment mutations write the sidecar file so agents can patch one card file without regenerating the whole ledger JSON.
