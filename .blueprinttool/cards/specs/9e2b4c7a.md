`ledger-cli unanswered --ledger <file>` must list every thread whose latest meaningful note is not `agent` or `assistant`, including thread id, target title, last note role/message/status/timestamp, and an exact `ledger-cli answer --ledger <file> --thread-id <id> --message ...` command. `ledger-cli answer` appends a persisted `role: agent` note to `ledger.notes[threadId]`, making the ledger a communication layer between operator and agent.
