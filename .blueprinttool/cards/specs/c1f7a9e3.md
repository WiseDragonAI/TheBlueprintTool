Voice recording captures its thread id at session start and every upload, transcription retry, and note update must use that captured thread id instead of the current selected card when async work finishes. Those updates stay operator-owned in the thread file. Card bodies and thread notes use the same markdown parser and renderer, including `---`, `***`, and `___` horizontal rules plus `##` and `###` heading blocks. Escaped newline sequences from CLI or agent answers are normalized into real markdown line breaks before block parsing. Thread inline code must not shrink below normal note text size, and operator note cards use the neutral input-grey surface with only a left accent border.
