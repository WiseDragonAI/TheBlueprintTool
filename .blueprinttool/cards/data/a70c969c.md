Runtime data used by generated functions.

- `State`: previous and next application data observed by reducers, controllers, components, and tests.
- `Action`: typed command created from input and consumed by controllers. Browser events never target DOM effects directly; they resolve an active document context and dispatch a controller-owned action.
- Route-addressable ledger tabs treat the server response as authoritative persisted state. Client actions may stage transient DOM feedback during gestures, but every persisted ledger mutation is committed through the active tab server route before replacing `activeLedger`.
